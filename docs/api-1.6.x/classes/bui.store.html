bui.store Class

Defined in: src/scripts/core/bui.store.js:10
Module: Core

Available since 1.5.1

数据行为存储器 Data Behavior Store

按vue接口设计的状态管理,但这不是vue,具体使用请查看教程或示例

预览地址: demo

bui.store

(
  • [option]
)

Defined in src/scripts/core/bui.store.js:10

Available since 1.5.1

Parameters:

  • [option] Object optional
    • [el] [最外层的选择器, 默认: ".bui-page"] String optional
    • scope String

      [数据的范围唯一标识, 比方公共数据可以用app, 模块的数据可以用page 或者其它自定义名称]

    • [isPublic] [是否作为公共数据, 默认: false ( 在模块里面使用) | true (在index.js 路由初始化以后使用)] Boolean optional
    • [needStatic] [1.5.1 静态解析{{page.title}}, 默认 false | true, 注意: isPublic:true 以后, needStatic不能为true, 在index.js把store挂载到路由,才能解析{{}},] Boolean optional
    • [needCompile] [1.5.1 默认编译对应的解析器] Boolean optional
    • [data] [ 动态监听的对象,当变更会自动触发相关联的选择器,需要提前定义] Object optional
    • [computed] [需要动态计算改变的相关联的时候,例如 fullName ,依赖于 data.firstName,data.lastName] Object optional
    • [watch] [ 监听的数据字段,在数据改变的时候,做对应的其它事情] Object optional
    • [methods] [ 定义的方法,便于拿取相关数据] Object optional
    • [mixins] [ 1.6.0新增 一个数组, 里面每个对象,都是一个单独组件,在当前实例合并,便于业务划分 {data:{},templates:{},methods:{},mounted:function(){}...} 参数跟store基本一致, 并且配合 view 标签,可以分模板处理] Array optional
    • [needView] [1.6.0 新增 是否解析view标签, 默认 true | false] Boolean optional
    • [needComponent] [1.6.0 新增 是否解析component标签, 默认 false(不解析) | true , 单页会由路由去解析] Boolean optional
    • [templates] [ 定义的模板方法] Object optional
    • [beforeMount] [ 数据解析前处理] Function optional
    • [beforeCompile] [1.5.3 新增 模板解析前处理] Function optional
    • [compiled] [1.5.3 新增 模板解析后处理] Function optional
    • [mounted] [ 数据解析完成以后处理] Function optional
    • [modelEvent] [默认: "input" | "change"] String optional

Example:

   var bs = bui.store({
                                     scope: "app",
                                     data: {
                                       size: 1
                                     }
                                   });
                                
                                   // 这样就会设置内容为1
                                   <b b-text="app.size"></b>
                                
                                   // 设置以后就会触发dom 的变更
                                   bs.size = 2;
