Methods

back

(
  • [option]
)
chainable

页面返回,支持回调,跟返回多层

Parameters:

  • [option] Object optional
    • [index] Number optional

      [默认:-1, 后退1层 ,负数,如果后退的层级大于历史记录,则退回到首页]

    • [name] Number optional

      [ 1.4.1新增 指定模块名称,该模块如果未存在,则后退一层]

    • [callback] Function optional

      [后退以后执行回调,回调里可以拿到后退模块的return值]

Example:

   方法1: 保持跟多页的开发方式一致

   bui.back();

   router.back();

   方法2: 后退上2页

   bui.back({ index: -2 });

   方法3: 后退以后刷新
   // 保持一致的后退刷新
   bui.back({ 
     callback: function() {
       // 如果回退到首页则刷新页面, 通过加载模块的方式,可以保持一致
       loader.require(["main"],function(main){
         // 刷新main
         main.init()
       })
     } 
   });
   或者
   router.back({ 
     callback: function() {
       router.refresh();
     } 
   });

   // 全局监听后退事件
   router.on("back",function (e) {
       // 如果回退到首页则刷新页面
         loader.require(["main"],function(main){
           // 刷新main
           main.init()
         })
   })

currentPage

() Object

Available since 1.4.2

获取当前页面下的DOM对象, 常用于当前模块事件绑定, 防止重复加载相同模块,导致事件重复绑定, 也适用于控件的重复

Returns:

Object:

[DOM对象]

Example:

     var currentPage = router.currentPage();

     // 绑定当前区域的按钮
     $(currentPage).on("click",".btn",function (e) {
       console.log(this);
     })

     // 当前区域的控件初始化
     var currentAccordion = $(".bui-accordion",currentPage);
     var uiAccordion = bui.accordion({
         id: currentAccordion
     });

load

(
  • [option]
)
chainable

页面跳转, 也支持定义好的模块跳转, 保持跟 bui.load一致

Parameters:

  • [option] Object optional
    • url String

      [页面跳转的地址,支持两种跳转方式, 一种是直接输入跳转的完整地址,一种是通过map方法注册页面key值,都要保持唯一]

    • param Object

      [传过去新页面的参数]

    • replace Boolean

      [是否是替换当前页面]

    • iframe Boolean

      [1.4.2新增支持iframe加载外部html]

Example:

 // 方法1: 保持跟多页的开发方式一致, 模块名为: pages/page2/page2

   bui.load({ url: "pages/page2/page2.html ", param: {} });

 // 方法2: 保持跟多页的开发方式一致, 模块名为: pages/page2/page2 

   router.load({ url: "pages/page2/page2.html", param: {} });

   });

option

(
  • [key]
  • [value]
)
chainable

获取设置参数

Parameters:

  • [key] String | object optional

    [ 不传则获取所有参数, 类型为string,没有第2个参数则获取某个参数 ]

  • [value] String | number | boolean | function optional

    [ 设置参数的时候要传,设置多个参数不用传,获取参数的时候也不用传 ]

Example:

     //获取所有参数
     var option = router.option();

     //获取某个参数
     var effect = router.option( "effect" );

     //修改一个参数
     router.option( "effect","cover" );

     //修改多个参数
     router.option( {"effect":"cover"} );

  • option

[预加载页面及资源,webapp预加载有助于切换优化 ]

Object:

     },{
       url: "pages/router/index.html"
   }]).then(function (res) {
       // 全部加载以后执行
   })

Events

off

Available since 1.3.0

为控件取消绑定事件

Event Payload:

  • [type] String optional

    [事件类型: "init" | "refresh"(刷新后触发) | "loadfail"(加载页面失败触发) | "load"(模块加载成功后触发,相同模板只加载一次) | "loadpart"(局部加载模块触发,相同模板只加载一次) | "back"(后退以后触发) | "complete"(页面完成时触发,load的时候就会触发) ]

  • [callback] Function optional

    [ 监听事件以后执行 ]

Example:

     router.off("refresh");

on

Available since 1.3.0

为控件绑定事件,以下事件均为全局事件. 全局事件建议写在index.js监听, 如果写在当前模块里面监听,当路由的参数cache=false时,需要先off掉,否则会有加载两次的,如果cache=true,相同模板只执行1次,则不需要off

Event Payload:

  • [type] String optional

    [ 事件类型: "init" | "refresh"(刷新后触发) | "load"(模块加载成功后触发,相同模板只加载一次) | "loadpart"(局部加载模块触发,相同模板只加载一次) | "back"(后退以后触发) | "complete"(页面完成时触发,load的时候就会触发) ]

  • [callback] Function optional

    [ 监听事件以后执行 ]

Example:

     router.on("refresh",function () {
         // 点击的菜单
         console.log(this);
     });
