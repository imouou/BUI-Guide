bui.pickerdate Class

Defined in: /Users/wilson/personal/htdocs/opensource/bui-rollup/src/scripts/ui/bui.pickerdate.js:7
Module: UI

日期选择器

日期必须以2015/8/10 这种格式传才能确保在IOS上正常, 支持动态渲染以及静态渲染, 两种的区别就在于,动态渲染初始化简单, 静态渲染灵活,支持自定义按钮样式,并且支持左右侧滑.

可以同时有左边跟右边菜单,菜单的个数也可以不一样多,留意示例的几种不同效果

预览地址: demo

方法说明:

value: 设置日期
min: 设置最小日期
max: 设置最大日期
reset: 重新渲染
onChange: 日期变更事件回调
formatValue: 数据格式化方法
cols: 设置分栏
widget: since1.3.0 获取依赖

      • month Object | "none"

      • date Object | "none"
        [ 不需要某栏,设置为 "none" 格式: { values: [],displayValues:[] } ]
      • hour Object | "none"

      • minute Object | "none"

    • [onInited] Function optional

      [ 1.5.1新增 初始化以后触发 ]

    • rotateEffect Boolean

      [ 是否开启3D效果,默认:false ]

    • onChange Function

      [ 回调 ]

    • callback Function

      [ 点击确定的回调 ]

Example:

// 1. 无底部按钮初始化

   var input = $("#datepicker_input");
   var uiPickerdate = bui.pickerdate({
       handle:"#datepicker_input",
       value: '2015/8/10 9:00',
       min: '2014/4/5 9:00',
       max: '2016/4/5 10:00',
       onChange: function(value) {
           input.val(value);
       }
   });

// 2. 带底部按钮的初始化

   var datePrevVal;
   var input = $("#datepicker_input");
   var uiPickerdate = bui.pickerdate({
       // id: "#datepicker_contanier",
       handle:"#datepicker_input",
       value: '2015/8/10 9:00',
       onChange: function(value) {
           input.val(value);
       },
       // 如果需要按钮,需要自己写callback
       buttons: ["取消","确定"],
       callback: function (e) {
           // 取消返回上一个值
           if( $(e.target).text().trim() == "取消"){
               uiPickerdate.value(datePrevVal);
           }else{
               datePrevVal = uiPickerdate.value();
           }
       }
   });

   // 获取上一个值
   datePrevVal = uiPickerdate.value();
